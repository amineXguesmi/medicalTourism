# Documentation Map

The requirements document is the source product brief. The architecture docs translate it into engineering boundaries and MVP delivery slices.

- `product/`: scope, backlog and original cahier des charges.
- `architecture/`: system shape, module boundaries, data model and API decisions.
- `decisions/`: ADRs for irreversible or expensive choices.
- `compliance/`: GDPR, clinical safety, AI risk and audit notes.
- `operations/`: environment and deployment planning.
